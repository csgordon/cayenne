Hidden arguments

Many functions have type arguments that seem to serve no purpose, except to irritate the user. E.g. 
if :: (a :: #) -> System$Bool.Bool -> a -> a -> a
For each use of if the type of the two branches must be given.

To lessen this problem Cayenne uses a mechanism for leaving out certain arguments at the application site. The arguments still have to be given when the function is defined though. Hidden agruments introduce a new version of the function type, function abstraction, and function application.

The function arrow in both the type and abstraction notation is written |-> for hidden arguments. Application of a hidden argument uses infix |. But normally a hidden argument does not need to be given at all.

In function definitions the hidden arguments should not be present on the left hand side unless preceded by a |, i.e., the left hand side looks like an application.

Example: 

if :: (a :: #) |-> Bool -> a -> a -> a;
if (True)  x y = x;
if (False) x y = y;
This if function can the be used as if True 1 2, or more explicitely if |Int True 1 2.

Hidden arguments is a syntactic device without any deep semantic properties. The function type for hidden arguments should not be viewed as a new type. It is completely compatible with the normal function type. It only servers as a marker to aid the insertion of the hidden argumets.

The hidden argument feature is still under development.
